# ОПИСАНИЕ ПРОЕКТА

## 1. Общая характеристика проекта
В рамках курсовой работы разрабатывается веб‑приложение и Telegram‑бот «Семейный круг» для организации взаимодействия между членами семьи. Система создаёт единое цифровое пространство для хранения информации о родственниках, отслеживания важных дат, планирования мероприятий и коммуникации.

Проект ориентирован на пользователей всех возрастов и не требует специальных технических знаний.

## 2. Актуальность проекта
Современный ритм жизни снижает частоту общения в семье: забываются важные даты, отсутствует совместное планирование, информация о родственниках хранится разрозненно. Существующие инструменты либо не ориентированы на семейные связи, либо не обеспечивают комплексного решения. «Семейный круг» объединяет хранение данных, планирование и коммуникацию в одной системе.

## 3. Цель и задачи проекта
### 3.1 Цель
Разработать веб‑приложение и Telegram‑бота, обеспечивающих удобное хранение данных о членах семьи, планирование совместных дел и автоматическое уведомление о важных событиях.

### 3.2 Задачи
- Проанализировать предметную область и существующие аналоги.
- Спроектировать архитектуру системы.
- Разработать структуру базы данных.
- Реализовать backend‑часть на Django.
- Создать пользовательский веб‑интерфейс.
- Разработать Telegram‑бота для уведомлений.
- Интегрировать веб‑приложение и Telegram‑бот.
- Провести тестирование системы.

## 4. Анализ существующих аналогов
### 4.1 Google Calendar
Поддерживает события и напоминания, но не хранит информацию о родственниках и не предоставляет семейный чат.

### 4.2 Мессенджеры (Telegram, WhatsApp)
Обеспечивают коммуникацию, но не имеют встроенного планирования задач и структурированной семейной базы.

### 4.3 Заметки и планировщики задач
Хранят задачи, но не ориентированы на совместное использование в семье.

**Вывод:** Аналоги не дают комплексного решения. «Семейный круг» объединяет хранение данных, планирование и коммуникацию в одной системе.

## 5. Используемые технологии
- **Язык:** Python
- **Фреймворк:** Django
- **База данных:** SQLite
- **Frontend:** HTML, CSS, JavaScript
- **Шаблоны:** Django Templates
- **Бот:** Telegram Bot API (python-telegram-bot)
- **Конфигурация окружения:** python-dotenv (.env)

## 6. Архитектура системы
Система состоит из двух компонентов:
- **Веб‑приложение** — центральный компонент, хранит данные, содержит бизнес‑логику и интерфейс.
- **Telegram‑бот** — предоставляет краткую информацию пользователям и поддерживает привязку аккаунта.

Также в проекте предусмотрен отдельный сервис (management command) для **плановой отправки напоминаний** в Telegram на основе настроек событий и задач.

## 7. Структурная схема системы
```
Пользователь
     |
     v
Веб‑браузер
     |
     v
Веб‑приложение (Django)
     |
     v
База данных (SQLite)
     ^
     |
Telegram‑бот
     |
     v
Telegram API

Планировщик напоминаний (Django management commands)
     |
     v
Telegram API
```

## 8. Проектирование базы данных
Реляционная база данных включает сущности:
- Пользователь
- Семья
- Участие в семье (роль пользователя в семье)
- Член семьи
- Событие
- Задача
- Пункты чек‑листа по задаче
- Вклады участников по задаче
- Сообщение
- Профиль Telegram (привязка аккаунта)
- Приглашение в семью

Структура обеспечивает логическое разделение данных и целостность информации.

## 9. Функциональные требования
### 9.1 Пользователи
- регистрация и авторизация;
- привязка Telegram‑аккаунта;
- управление профилем.

### 9.2 Семьи и доступ
- создание семьи;
- просмотр семьи и списка участников;
- роли доступа: владелец / администратор / участник;
- приглашение пользователя в семью по никнейму (username) и принятие приглашения.

### 9.3 Члены семьи
- добавление и редактирование информации;
- отображение родственных связей.

Дополнительно реализована визуализация семейного дерева с настройкой связей (родители, супруги) и управлением тем, какие участники отображаются в дереве.

### 9.4 События
- добавление дней рождения и праздников;
- календарь событий;
- напоминания.

В карточке события можно указать тип события, привязать его к конкретному члену семьи и настроить напоминание за N дней до даты.

### 9.5 Задачи
- создание и распределение задач;
- отслеживание выполнения;
- финансовое участие (вклады и расходы по задачам);
- чек‑лист исполнения задач.

Поддерживается срок выполнения, статус (нужно сделать / в процессе / готово), бюджет и напоминание за N дней до срока.

### 9.6 Коммуникация
- семейный чат;
- хранение истории сообщений.

Сообщения разделены по семьям; удаление доступно отправителю, либо администратору/владельцу семьи.

### 9.7 Telegram‑бот
- привязка аккаунта Telegram к пользователю сайта с подтверждением;
- просмотр списка семей;
- просмотр ближайших событий и задач;
- просмотр последних сообщений;
- команда «на сегодня» (события/задачи текущей даты).

Отправка напоминаний о событиях и задачах выполняется отдельной командой Django (может запускаться периодически планировщиком).

## 10. Нефункциональные требования
- простой и интуитивно понятный интерфейс;
- адаптивность под разные устройства;
- надёжное хранение данных;
- минимальные требования к обучению пользователей;
- конфиденциальность данных семьи.
- веб‑приложение и Telegram‑бот должны работать как совместно, так и автономно.

Ограничения учебного проекта: используется SQLite и локальная конфигурация окружения через переменные (например, токен Telegram‑бота).

## 12. Ожидаемый результат
Создание веб‑приложения и Telegram‑бота «Семейный круг», позволяющих хранить семейную информацию, отслеживать важные даты, планировать мероприятия и улучшать коммуникацию между родственниками.

## 13. Область применения и пользователи
Приложение предназначено для семейных групп, которым нужен удобный инструмент для хранения контактов, отслеживания дат, планирования совместных активностей и коммуникации.

## 14. Технические данные и условия эксплуатации
- **Платформа:** веб‑приложение (браузер) + Telegram‑бот
- **ОС пользователя:** Windows / macOS / Linux, мобильные ОС с браузером
- **Браузеры:** актуальные версии Chrome, Firefox, Safari, Edge
- **Серверная часть:** Python 3.x, Django, SQLite (учебная версия)
- **Сетевые требования:** доступ к интернету

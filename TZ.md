# ТЕХНИЧЕСКОЕ ЗАДАНИЕ (ТЗ)

## 1. Наименование проекта
Веб‑приложение и Telegram‑бот «Семейный круг».

## 2. Основание для разработки
Курсовая работа по разработке информационной системы для организации семейного взаимодействия.

## 3. Цель разработки
Создать систему (веб‑приложение + Telegram‑бот), позволяющую:
- вести список семей и участников;
- хранить данные о родственниках;
- вести семейные события и задачи;
- поддерживать семейный чат;
- получать напоминания о событиях и задачах в Telegram.

## 4. Объект автоматизации
Бытовое взаимодействие членов семьи: хранение информации, планирование событий и задач, коммуникация.

## 5. Термины и сокращения
- **Семья** — группа пользователей, объединённых общим пространством данных.
- **Участник семьи** — пользователь, включённый в семью (роль owner/admin/member).
- **Родственник** — запись о человеке в семье (может быть не пользователем сайта).
- **Событие** — важная дата (день рождения/праздник/семейное/другое).
- **Задача** — планируемое действие с ответственным, сроком, статусом.

## 6. Функциональные требования (фактически реализовано)

### 6.1 Авторизация и регистрация
- Пользователь может зарегистрироваться (username, ФИО, пароль).
- Пользователь может войти в систему.

Критерии приемки:
- После регистрации пользователь автоматически авторизован и попадает на главную страницу.

### 6.2 Семьи и доступ
- Пользователь может создать семью (название, описание).
- Создатель становится **владельцем (owner)** семьи.
- Для семьи ведётся список участников (membership) с ролями:
  - **owner** — владелец (может управлять ролями и удалять семью);
  - **admin** — администратор;
  - **member** — участник.
- Владелец может менять роли участников; поддерживается передача роли owner другому участнику.
- Владелец/администратор может создать приглашение в семью по `username`.
- Пользователь может принять приглашение (после входа на сайт).
- Владелец/администратор может отозвать приглашение.
- Владелец/администратор может удалить участника из семьи (кроме владельца).

Критерии приемки:
- Пользователь без доступа к семье не видит её данные.
- Нельзя удалить владельца из семьи.

### 6.3 Родственники (карточки членов семьи)
- В семье можно создавать записи родственников, не привязанные к аккаунту пользователя.
- Поля родственника (основные): ФИО, родство, дата рождения, контакты, адреса, соцсети, место работы, заметки.
- Поддержаны ссылки на родителей: `parent1`, `parent2`.
- Реализованы операции: добавить / редактировать / удалить.

Критерии приемки:
- Пользователь видит список родственников только текущей семьи.

### 6.4 Семейное дерево
- Отдельная страница отображает «дерево» (визуализация на основе данных членов семьи).
- Есть настройка отображения в дереве через признак `in_tree`.
- Реализовано редактирование связей дерева (доступ owner/admin):
  - связать супругов (pair) и разорвать связь (unpair);
  - назначить родителей ребёнку (add_child/set_parent);
  - очистить родителей (clear_parents);
  - исключить участника из дерева (remove_from_tree);
  - очистить дерево целиком (clear_tree) и показать всех (show_all);
  - изменить порядок отображения (order update).

Критерии приемки:
- Доступ к редактированию дерева имеет только owner/admin.

### 6.5 События
- Пользователь может добавлять события в рамках семьи:
  - название, дата, тип (birthday/holiday/family/other), описание;
  - привязка к родственнику (опционально);
  - настройка напоминания: `remind_days_before`.
- Реализованы операции: добавить / редактировать / удалить.

Критерии приемки:
- События фильтруются по выбранной семье.

### 6.6 Задачи
- Пользователь может добавлять задачи в рамках семьи:
  - название, описание, статус (todo/in_progress/done);
  - срок `due_date` (опционально);
  - напоминание за N дней `remind_days_before`;
  - ответственный `assignee` (из участников семьи);
  - бюджет `budget`.
- Реализованы операции: добавить / редактировать / удалить.
- Детальная страница задачи:
  - чек‑лист (TaskChecklistItem): добавление пунктов и отметка выполнения;
  - вклады участников (TaskContribution): user, amount, comment;
  - расчёт суммарных вкладов (total_cost) как сумма всех contributions.

Критерии приемки:
- Ответственный выбирается только из пользователей, состоящих в семье.

### 6.7 Сообщения (семейный чат)
- Для каждой семьи доступна лента сообщений.
- Пользователь может отправлять сообщения.
- Удаление сообщения:
  - отправитель может удалить своё сообщение;
  - owner/admin может удалить чужое сообщение.

Критерии приемки:
- Сообщения одной семьи не видны участникам другой семьи.

### 6.8 Привязка Telegram и бот
#### 6.8.1 Привязка
- Пользователь может привязать Telegram через бота командой `/link <username>`.
- Привязка требует подтверждения на сайте по одноразовой ссылке вида `/telegram/confirm/<token>/`.
- Поддержаны команды: `/unlink`, `/me`.

Критерии приемки:
- Без подтверждения привязки бот не выдаёт данные (сообщает о необходимости подтверждения).

#### 6.8.2 Команды бота (после подтверждения)
- `/families` — список семей пользователя.
- `/events [дней]` — ближайшие события (по умолчанию 30, максимум 180).
- `/tasks [дней]` — активные задачи (не done) с ближайшими сроками.
- `/messages` — последние сообщения.
- `/today` — события/задачи на текущую дату.

## 7. Напоминания в Telegram (плановая отправка)
- Реализована команда `send_reminders`, которая отправляет напоминания пользователям с подтверждённым Telegram‑профилем.
- Напоминания формируются:
  - по событиям: если `event.date - remind_days_before == today`;
  - по задачам: если `task.due_date - remind_days_before == today` и задача не завершена.
- Реализована команда `run_reminder_scheduler`, запускающая `send_reminders` циклически с интервалом `REMINDER_INTERVAL_SECONDS` (по умолчанию 86400 секунд).

## 8. Нефункциональные требования
- Интерфейс: простой, web‑ориентированный, доступный без спец. навыков.
- Конфиденциальность: доступ к данным семьи только у её участников.
- Надёжность: хранение данных в SQLite (учебный вариант), операции CRUD с валидацией форм.

## 9. Требования к окружению и конфигурации
- Python 3.x.
- Зависимости (см. `requirements.txt`): Django, python-telegram-bot, python-dotenv.
- Переменные окружения:
  - `TELEGRAM_BOT_TOKEN` — токен бота (обязателен для бота и напоминаний);
  - `SITE_URL` — базовый URL сайта для формирования ссылки подтверждения (по умолчанию `http://localhost:8000`);
  - `REMINDER_INTERVAL_SECONDS` — интервал планировщика (по умолчанию 86400).

## 10. Состав и структура интерфейса (основные страницы)
- Главная (дашборд): текущая семья, ближайшие события/задачи, последнее сообщение, статус приглашений/привязки Telegram.
- Семьи: создание, список семей.
- Карточка семьи: участники, роли, приглашения.
- Родственники: список, добавление/редактирование.
- Семейное дерево: просмотр и редактирование.
- События: список, добавление/редактирование.
- Задачи: список, добавление/редактирование, детальная страница.
- Сообщения: лента сообщений семьи.
- Страница подтверждения Telegram.

## 11. Порядок контроля и приемки
Работа считается выполненной при выполнении критериев приемки из раздела 6 и при возможности:
- запустить веб‑приложение и выполнить основные сценарии (семья → участники → события/задачи → сообщения);
- запустить бота и получить данные после подтверждения привязки;
- отправить напоминания командой `send_reminders` (при наличии данных и подтверждённой привязки).
